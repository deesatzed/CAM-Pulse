# cag-core

Standalone Cache-Augmented Generation (CAG) runtime for loading and injecting pre-built knowledge corpora into LLM prompts.

Part of the [CAM-Pulse](https://github.com/deesatzed/CAM-Pulse) project.

## What is CAG?

Cache-Augmented Generation is a pattern where a pre-built knowledge corpus is loaded into the LLM context window at inference time. Unlike RAG (Retrieval-Augmented Generation), which retrieves relevant chunks per query, CAG loads the entire corpus once and relies on the LLM's attention mechanism to find relevant information. This approach:

- Eliminates retrieval latency
- Avoids embedding model dependencies at runtime
- Enables local LLM KV prefix caching for repeated queries
- Requires zero external dependencies (stdlib only)

```bash
pip install cag-core
```

## Quick Start

### 1. Prepare your corpus files

You need two files in a directory:

- `corpus.txt` -- the serialized knowledge text
- `meta.json` -- metadata about the corpus

```json
{
  "ganglion": "my-project",
  "methodology_count": 42,
  "built_at": "2026-03-31T12:00:00Z",
  "stale": false,
  "corpus_tokens_approx": 8000,
  "pointer_count": 0,
  "shorthand_compression": false
}
```

These files are typically generated by `cam cag convert --export-recipe`, but you can create them manually or with any pipeline.

### 2. Load and inject into OpenAI-style messages

```python
from cag_core import CAGCache, inject_into_messages

cache = CAGCache(cache_dir="./my_corpus")
if cache.load():
    corpus = cache.get_corpus()

    messages = [{"role": "user", "content": "How do I implement retry logic?"}]
    inject_into_messages(corpus, messages)

    # messages[0] is now a system message containing the knowledge base
    # messages[1] is the original user message
    # Pass messages to your LLM API of choice
```

### 3. Build a stable system message for local LLM KV caching

```python
from cag_core import CAGCache, build_system_message

cache = CAGCache(cache_dir="./my_corpus")
cache.load()

sys_msg = build_system_message(cache.get_corpus())

# Use as the system message in your LLM calls.
# The preamble is fixed, so local LLM servers (Ollama, llama.cpp)
# can cache the KV state and skip re-computation on subsequent requests.
messages = [
    {"role": "system", "content": sys_msg},
    {"role": "user", "content": "How do I implement retry logic?"},
]
```

### 4. Check cache status

```python
from cag_core import CAGCache

cache = CAGCache(cache_dir="./my_corpus")
cache.load()

status = cache.get_status()
print(f"Ganglion:      {status['ganglion']}")
print(f"Loaded:        {status['loaded']}")
print(f"Stale:         {status['stale']}")
print(f"Methodologies: {status['methodology_count']}")
print(f"Approx tokens: {status['corpus_tokens_approx']:,}")
```

### 5. Mark cache as stale

When your knowledge base changes, mark the cache so it gets rebuilt:

```python
from cag_core import CAGCache

cache = CAGCache(cache_dir="./my_corpus", ganglion="my-project")
cache.mark_stale()
# meta.json now has "stale": true
```

## API Reference

### `CAGCache(cache_dir=".", ganglion="imported")`

Load and serve a pre-built CAG corpus.

- **`load() -> bool`** -- Read corpus.txt and meta.json from disk
- **`is_loaded() -> bool`** -- Check if corpus is in memory
- **`is_stale() -> bool`** -- Check if cache is marked stale
- **`get_corpus() -> str`** -- Return the loaded corpus text
- **`get_status() -> dict`** -- Return cache status dict
- **`mark_stale() -> None`** -- Set stale flag on disk
- **`corpus_hash() -> str`** -- 12-char MD5 prefix for tracking

### `inject_into_messages(corpus, messages, knowledge_budget=16000)`

Inject corpus into an OpenAI-style message list in place. Appends to existing system message or inserts a new one at position 0.

### `build_system_message(corpus, knowledge_budget=16000)`

Return a stable system message string with fixed preamble for KV prefix caching.
